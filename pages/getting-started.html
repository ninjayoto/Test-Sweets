What is Test Sweets?

Test sweets is a tool I created for myself to quickly setup automated tests for front-end Javascript. It is a solid foundation of tools and plumbing that can drop into an existing project or developed alongside a new one. It is also the result of lots of my own research that has helped me understand the core language and made me a better programmer.

What Test Sweets is not.

Test sweets is not a testing framework. It uses a test framework called qUnit, made by the team that makes jQuery.

There are other testing frameworks available like Mocha, Jasmine, or Buster. Test Sweets is compatible with those frameworks, but not out of the box.

Why use Test Sweets?

  • It helps automate a very boring and tedious task
  • It gives you constant immediate feedback on the state of your application
  • Create an early warning system that catches the worst kind of bugs: silent failures
  • Possess greater sense of confidence when making changes to your codebase
  • To have a goal in life.

Handy Features:

  • qUnit assertions cheat sheet
  • A sample test runner for rapid cross browser testing, even in virtual machines and mobile SDKs
  • Sample Configurations for integrating Test Sweets into your Grunt tasks
  • Testing at the command line using PhantomJS

Before we get started, these are two tools I assume you have heard of and know how to use:

  • GruntJS, the automated task runner for the web.
  • NPM, Node Package Manager for downloading and installing dependencies

If you have not heard of or used either of these tools, I highly suggest Chris Coyier's Grunt for People Who Think Things Like Grunt are Weird and Hard.
